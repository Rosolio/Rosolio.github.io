## 1. 算法的复杂度与最优性（概念 + 举例）

- **最坏复杂度 (Worst-case)**：输入规模为 `n` 时，算法在所有可能输入下**运行时间的最大值**。  
  *例*：快排最坏为 `O(n²)`（有序数组且固定选首元素）。
- **平均复杂度 (Average-case)**：输入规模为 `n` 时，在**所有可能输入等概率出现**下运行时间的期望值。  
  *例*：快排平均为 `O(n log n)`。
- **最优性 (Optimality)**：若一个算法的**最坏复杂度**等于该问题的**理论下界**，则称该算法为（最坏意义下）最优的。  
  *例*：比较排序下界为 `Ω(n log n)`，归并排序达到该下界，是理论最优；快排因最坏 `O(n²)`，非最坏最优。

## 2. 渐近符号 `o`、`Ω`、`Θ` 的定义与证明

设 `f(n)` 和 `g(n)` 为正函数，且 `n > n₀`：

| 符号          | 定义（数学形式）                                            | 口语含义         |
|:------------|:----------------------------------------------------|:-------------|
| **Θ(g(n))** | 存在正常数 `c₁, c₂, n₀`，使 `0 ≤ c₁·g(n) ≤ f(n) ≤ c₂·g(n)` | **紧界**（同阶）   |
| **Ω(g(n))** | 存在正常数 `c, n₀`，使 `0 ≤ c·g(n) ≤ f(n)`                 | **下界**（不低于）  |
| **O(g(n))** | 存在正常数 `c, n₀`，使 `0 ≤ f(n) ≤ c·g(n)`                 | **上界**（不高于）  |
| **o(g(n))** | 对任意正常数 `c`，存在 `n₀`，使 `0 ≤ f(n) < c·g(n)`            | **非紧上界**（低阶） |

**证明示例**：证明 `2n² + 3n = Θ(n²)`  
取 `c₁=2, c₂=4, n₀=3`，有 `2n² ≤ 2n²+3n ≤ 4n²`（对 `n≥3` 恒成立）。

## 3. 二阶常系数齐次递推：`aₙ = r₁·aₙ₋₁ + r₂·aₙ₋₂` 完整求解

- **Step 1**：构造**特征方程**：`x² - r₁·x - r₂ = 0`。
- **Step 2**：求特征根 `x₁, x₂`，分三类讨论：

| 根的情况                      | 通解形式                                                                  |
|:--------------------------|:----------------------------------------------------------------------|
| **两不等实根** (`x₁ ≠ x₂`)     | `aₙ = A·x₁ⁿ + B·x₂ⁿ`                                                  |
| **二重实根** (`x₁ = x₂ = x₀`) | `aₙ = (A + B·n)·x₀ⁿ`                                                  |
| **共轭复根** (`α ± βi`)       | `aₙ = ρⁿ·(C₁·cos nθ + C₂·sin nθ)` <br>其中 `ρ = √(α²+β²)`, `tanθ = β/α` |

- **Step 3**：代入初始值（如 `a₀, a₁`），解方程组定系数 `A, B`（或 `C₁, C₂`）。

## 4. 平滑函数性质 + 主定理求解 `T(n) = b·T(n/c) + f(n)`

> **注意**：标准写法通常为 `T(n) = aT(n/b) + f(n)`，此处按你提纲的 `b, c` 书写，基准为 `n^(log_c b)`。

- **平滑函数性质（正则性条件）** ：用于**情况 3**的校验。若 `f(n)` 满足存在常数 `λ < 1`，使得 `b·f(n/c) ≤ λ·f(n)`（`n` 足够大），则称 `f(n)` 是平滑的。

**主定理三情况**（比较 `f(n)` 与 `n^(log_c b)`，令 `E = log_c b`）：

| 情况    | 条件（存在 `ε > 0`）                 | 结论                                 |
|:------|:-------------------------------|:-----------------------------------|
| **1** | `f(n) = O(n^(E - ε))` （多项式地小）  | `T(n) = Θ(n^E)`                    |
| **2** | `f(n) = Θ(n^E · log^k n)` （同阶） | `T(n) = Θ(n^E · log^(k+1) n)`      |
| **3** | `f(n) = Ω(n^(E + ε))` （多项式地大）  | **必须验证平滑条件**，若成立则 `T(n) = Θ(f(n))` |

**失效警告**：若 `f(n)` 不满足上述三种严格倍数差距（如落在间隙中），或情况3不平滑，主定理失效，需改递归树法。

## 5. 堆的定义、数组表示、基础操作

- **定义**：逻辑上是**完全二叉树**。**最大堆**：父 ≥ 子；**最小堆**：父 ≤ 子。兄弟间无大小关系。
- **数组表示（下标从 0 起）** ：
  - `Parent(i) = (i-1)//2`
  - `Left(i) = 2*i + 1`
  - `Right(i) = 2*i + 2`
    *(若下标从1起，则分别为 `i//2`, `2i`, `2i+1`)*

| 操作                  | 核心逻辑                                              | 时间复杂度            |
|:--------------------|:--------------------------------------------------|:-----------------|
| **上浮 (Shift Up)**   | 插入末尾，不断与父交换                                       | `O(log n)`       |
| **下沉 (Shift Down)** | 删除堆顶时：交换到末尾删除，新顶与**较小/较大**孩子交换                    | `O(log n)`       |
| **取堆顶**             | 返回 `arr[0]`                                       | `O(1)`           |
| **建堆 (Heapify)**    | 从 **最后一个非叶子节点** `(n//2 - 1)` 开始，**从右向左**对每个节点执行下沉 | **`O(n)`** （重点！） |

## 6. 复用快排平均复杂度分析方法（处理递推式）

对于递推式：
`A(n) = (n-1) + (2/n) · Σ_{i=1}^{n-1} A(i)`

- **含义**：快排的平均比较次数。`(n-1)` 是单次分区比较次数，`(2/n)Σ` 是左右子问题平均规模的和。
- **求解技巧（差分法）** ：
  1. 两边同时乘以 `n`：`n·A(n) = n(n-1) + 2·Σ_{i=1}^{n-1} A(i)`
  2. 代入 `n-1` 得：`(n-1)·A(n-1) = (n-1)(n-2) + 2·Σ_{i=1}^{n-2} A(i)`
  3. 两式相减消去求和，化简得：`A(n)/(n+1) = A(n-1)/n + 2(n-1)/(n(n+1))`
  4. 累加（迭代）得：`A(n) = O(n log n)`

**应用**：该数学工具可用于分析**随机 BST（二叉搜索树）** 的平均查找长度、随机选择算法的期望复杂度。

## 7. 对抗策略（对手论证）分析经典问题

- **核心思想**：设计一个“对手”动态给出输入数据的比较结果，迫使算法走最坏路径，从而推导出问题的**下界**。

| 经典问题         | 对抗策略与结论                                                                                                |
|:-------------|:-------------------------------------------------------------------------------------------------------|
| **同时求最大和最小** | 两两配对比较。先两两比，较小者进“小池”争最小，较大者进“大池”争最大。<br>**下界**：`⌈3n/2⌉ - 2` 次比较（最优算法恰好达到）。                             |
| **求第二大元素**   | 采用**锦标赛树**。先构建淘汰赛找出最大值（`n-1` 次比较），最大值打败过的元素（共 `⌈log₂n⌉` 个）中，最大的即为第二大。<br>**下界**：`n + ⌈log₂n⌉ - 2` 次比较。 |

## 8. 线性时间选择算法（BFPRT / 中位数的中位数）

- **目标**：在无序数组中找到第 `k` 小的元素，最坏时间复杂度 **`O(n)`**。
- **实现步骤**：
  1. 将数组分成 `⌊n/5⌋` 组，每组 5 个元素（最后一组不足 5 个）。
  2. 对每组进行排序，找出每组的中位数。
  3. 递归调用本算法，找到所有组中位数的中位数，记为 `pivot`。
  4. 用 `pivot` 对原数组进行 **PARTITION**（分区），得到 `pivot` 的排名 `m`。
  5. 若 `k == m`，返回 `pivot`；若 `k < m`，递归处理左半边；否则递归处理右半边。
- **应用**：解决 Top-K 问题、数值中位数查找，用于优化快排避免最坏情况。

## 9. 平摊分析（Amortized Analysis）与经典问题

- **核心思想**：求一系列操作的平均代价，允许单个操作昂贵，但保证**总代价**有紧上界。
- **三种方法**：聚合分析、记账法（会计法）、势能法。

| 经典问题                 | 平摊结论与分析思路                                                                                                                                    |
|:---------------------|:---------------------------------------------------------------------------------------------------------------------------------------------|
| **MultiPop 栈**       | 支持 `Push`（1单位）和 `MultiPop(k)`（弹出k个）。<br>每个元素**最多被弹出一次**。`n` 个操作总代价为 `O(n)`，**平摊 `O(1)`**。                                                    |
| **DeleteLargerHalf** | 数据结构删除较大的一半元素。使用**势能法**，设势能为当前元素个数，每次操作后势能骤降，**平摊 `O(1)`**。                                                                                  |
| **栈模拟队列**            | 用两个栈 `inbox` 和 `outbox`。<br>`Enqueue` 直接压入 `inbox`（O(1)）。<br>`Dequeue` 时若 `outbox` 空，则将 `inbox` 全部弹出压入 `outbox`（摊还至每个元素只移动一次）。**平摊 `O(1)`**。 |

## 10. BFS / DFS 思想及图上经典问题

| 算法             | 核心思想（数据结构）          | 特点                                  |
|:---------------|:--------------------|:------------------------------------|
| **DFS (深度优先)** | 递归 / 栈。沿着一条路走到黑，回溯。 | 记录**发现时间**和**完成时间**，适合求连通性、拓扑序、SCC。 |
| **BFS (广度优先)** | 队列。层层向外扩散。          | 天然适合求**无权最短路**、层级遍历。                |

**四类经典问题解法**：

| 问题              | 应用算法                   | 核心要点                                         |
|:----------------|:-----------------------|:---------------------------------------------|
| **拓扑排序 (Topo)** | DFS 或 BFS（Kahn）        | 有向无环图（DAG）。DFS：按**完成时间逆序**输出；Kahn：不断删入度为0的点。 |
| **任务调度**        | Topo 排序 + DP           | 在拓扑序上递推最早/最晚完成时间。                            |
| **关键路径**        | DFS / DP（DAG最长路）       | 先拓扑排序，按序松弛 `dis[v] = max(dis[u] + w)`。       |
| **强连通分量 (SCC)** | DFS（Kosaraju 或 Tarjan） | Kosaraju：正图 DFS 记顺序，反图按逆序 DFS 找 SCC。         |

## 11. 贪心法（思想、特点、经典问题与证明）

- **思想**：每一步都做出当前看起来**最优**（局部最优）的选择，希望得到全局最优。
- **特点**：**无后效性**（后续决策不影响已做决策）且问题具有**最优子结构**。
- **证明思路**（最关键）：**交换论证法**（Exchange Argument）或**归纳法**。假设存在一个最优解，将其第一个元素与贪心选择交换后，解不会变差。

| 经典问题                 | 贪心策略                        | 正确性证明思路                                                  |
|:---------------------|:----------------------------|:---------------------------------------------------------|
| **换硬币**（标准面额）        | 每次尽可能取最大面额。                 | 标准面额下最优；但贪心**并不通用**（如面额1, 3, 4，凑6贪心得4+1+1=3枚，最优为3+3=2枚）。 |
| **单源最短路径（Dijkstra）** | 每次从**未确定**集合中选取距离源点最近的顶点加入。 | 反证法：若经过其它未确定点再到达该点，路径长度必然大于当前直连距离（前提：**边权非负**）。          |

## 12. 动态规划（思想、特点、经典问题递归式）

- **思想**：通过**最优子结构**（大问题依赖子问题的最优解）和**重叠子问题**（记忆化避免重复），自底向上求解。
- **特点**：暴力枚举所有可行状态，但利用递推避免指数爆炸。

| 经典问题              | 子问题递推式（状态转移方程）                                                                                                   | 备注                  |
|:------------------|:-----------------------------------------------------------------------------------------------------------------|:--------------------|
| **最长公共子序列 (LCS)** | `dp[i][j] = dp[i-1][j-1] + 1` (若 `a[i]==b[j]`)<br>`dp[i][j] = max(dp[i-1][j], dp[i][j-1])` (否则)                  | 时间复杂度 `O(nm)`       |
| **抽牌拿取最大值（区间DP）** | 设 `dp[i][j]` 为当前玩家在 `[i, j]` 区间能拿到的最大分数（对手最优）。<br>`dp[i][j] = max( nums[i] - dp[i+1][j], nums[j] - dp[i][j-1] )` | 最终结果为 `dp[0][n-1]`  |
| **回文分割（最小分割数）**   | 预处理 `isPal[i][j]`。<br>`dp[i]` 表示前缀 `[0..i]` 的最少分割数。<br>`dp[i] = min( dp[j-1] + 1 )` 对 `isPal[j][i] == True`      | 初始化 `dp[i]=i`（最坏切分） |
